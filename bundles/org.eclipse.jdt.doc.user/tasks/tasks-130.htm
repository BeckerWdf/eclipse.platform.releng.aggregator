Launching a Java program

Java programs may be launched in either run or debug mode.

  • In run mode, the program executes, but the execution may not be suspended or examined.
  • In debug mode, execution may be suspended and resumed, variables may be inspected, and expressions may be evaluated.

To launch a Java program:

  1. In the Package Explorer view, select the Java compilation unit or class file you want to launch.

  2. If you want to debug the program first, click the Debug button in the workbench toolbar. Otherwise, click the Run button.

  3. If you have not already set a launcher for the project, you will be prompted for a launcher to use.
    Note: You can set the selected launcher as the default for future launches in the project by selecting the Set as default launcher for project check box.

  4. If you selected a Java program, your program is now launched in the mode you selected, and the launch appears in either the Debug view (if launched in debug mode) or the Processes view (if launched in run mode).

    If you selected a Java project, you are prompted to select the class you wish to launch. Only those classes that define a main method are available to select. The chosen class is launched in the mode you selected.

Launching from other views and editors

You can launch a program without first selecting it in the Packages or Navigator view. For example, if the active editor is open on the program you wish to launch, you can click the Run or Debug button.

Launching with a non-default launcher

You may also launch a program using a different launcher without having to change the default launcher for a project.

To do this, select a program or project in the Package Explorer view, then open the Run or Debug button pull-down menus to see a list of all launchers available for the chosen mode.
