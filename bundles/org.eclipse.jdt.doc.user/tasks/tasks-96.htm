Renaming a method

Renaming a method declared in an interface also renames (and updates all references to) all methods that are implementations of that method.

When renaming a non-static method declared as public, package visible, or protected, all methods overriding it are also renamed and all references to them updated.

Renaming a static method or a private method updates references only to that method.

You can rename a method by modifying its declaration in the compilation unit in which it is declared. However, if you also want to update all references to it, you must either:

  • In the Packages view, go to the compilation unit where the method is declared, and double-click it to open it in the editor. Select the method in the Outline view, from its pop-up menu, select Refactor > Rename.
  • In the Packages view, go to the compilation unit where the method is declared, and from its pop-up menu, select Open Type Hierarchy. Select the method in the Hierarchy view and select from its pop-up menu Refactor > Rename.
