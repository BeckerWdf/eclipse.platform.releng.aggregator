Building and Fixing Problems in Your Code

In this section, you will build Java projects and fix problems.
 
1.
Verify that there are currently no problems in the workbench by looking in the Tasks view, where build problems are listed if they exist.
2.
Open junit.framework.TestCase.java in the editor by double clicking the file in the Package Explorer.
3.
Add a syntax error by deleting the semicolon at the end of the first line in the file ( package junit.framework; ).
4.
Click the Save button when you are done. The project is incrementally built, and the problem is discovered.
Note: Only the changed file and the file depending on it are recompiled.
5.
The problem is indicated as follows:
  • In the Tasks view, the problems are listed.
  • Problem ticks show up on affected Java elements and their parent elements in the Package Explorer, the type hierarchy or the outline view.
  • In the editor, a problem marker is displayed near the affected line.
  • In the editor tab shows a problem marker.
6.
Hover over the problem marker in the marker bar in the editor area and view the information in the hover help.
7.
Click the Close button on the editor's tab to close the editor.
8.
In the Tasks view, select the problem in the list. Open its context menuand select Go To. The file where the problem is detected opens in the editor area, where the problem is shown.
9.
Correct the problem in the editor by adding the semicolon, then click the Save button. 

The project is built automatically on save, and all problem indicators in the workbench disappear.
10.
 In the outline view select the method getName() to scroll to this method.
11.
 On the first line of the method change the returned variable fName to fTestName. While you type, a problem indication line is shown in the editor to inform about a syntax or semantic problem.
12.
 On the left border of the editor a light bulb marker shows up. The light bulb signals that a correction proposal is available.
13.
 Set the cursor inside the marked range and choose Quick Fix from the Edit menu bar. You can also press CTRL + 1 or left click the light bulb. A selection dialog shows up proposing possible corrections.
14.
 Select 'Change to fName' to fix the problem. The problem indication line will go away as the correction is applied.
15.
 Close the file without saving.
16.
 You can enable or disable the problem indication feature under Window > Preferences > Java > Editor > Problem Indication
