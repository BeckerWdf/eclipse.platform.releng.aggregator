Java Preferences

Option

Description

Default

Link Package Explorer selection to active editor

Choose to turn this option on or off.

  • If this option is turned on, then when an open resource is selected in the Package Explorer view, the editor containing that resource becomes the active editor in the editor area. Similarly, when you click in an editor in the editor area, the corresponding resource is selected in the Package Explorer view.
  • If this option is turned off, then selecting an open file in the Package Explorer view has no effect on the editor area. In order to make an editor active, you will have to either click the editor's tab or double-click the resource in the Package Explorer view.

On

Action on double-clicking in the Package Explorer on containers is "Go Into"

When this option is turned on, then when you double-click a container, a Go Into command is executed.

See Go Into

Off

Update Java views

On save only:
Java elements displayed in all views except the Outline view are not updated until a compilation unit is saved. The views reflect the current state of the workspace, without consideration of working copies.
While editing:
Java elements displayed in all views always reflect the current state of the workspace, including working copies.
