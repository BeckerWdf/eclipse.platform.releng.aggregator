Hierarchy view

This view shows a type's hierarchy in a different presentation

Back

This command returns to the most previously-displayed type hierarchy.

Back Button

Forward

This command returns to the type that was viewed immediately after the currently-displayed type hierarchy.

Forward Button

Type hierarchy

This command displays the type in its full context (i.e., superclasses and subclasses) in the Hierarchy view. To see for which type the hierarchy is shown, hover over the view title (e.g., "Types").

Type Hierarchy

Supertypes hierarchy

This command displays the supertypes and the hierarchy of all implemented interfaces of the type in the Hierarchy view. The tree starts at the selected type and displays the result of traversing up the hierarchy.

Note: The selected type is always at the top level, in the upper-left corner.

Supertypes Hierarchy

Subtypes hierarchy

This command displays the subtypes of the selected class and/or all implementers of the interface in the Hierarchy view. The tree starts at the selected type and displays the result of traversing down the hierarchy

Note: The selected type is always at the top level, in the upper-left corner.

List pane commands

Button

Command or Option

Description

Lock View and Show Members in Hierarchy/
Unlock View

Shows the members implementing the selected method in the list pane in the type hierarchy. Only types implementing the method are shown.

When the view is locked, the list pane no longer tracks the selection in the hierarchy pane above.

Show/Hide All Inherited Members

Shows or hides all methods and fields inherited by base classes. When this option is set to Show, the name of the type that defines the method is appended to the method name.

Show/Hide Fields

Show or hide the fields in the list view below the type hierarchy.

Show/Hide Static Members

Show or hide the static fields and methods in the list view below the type hierarchy.

Show/Hide Non-Public Members

Show or hide the static fields and methods in the list view below the type hierarchy.
