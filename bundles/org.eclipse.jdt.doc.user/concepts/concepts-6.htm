Java views

The Java development tools contribute the following views to the workbench:

Package Explorer

The Package Explorer that is shown by default in the Java perspective shows the Java element hierarchy of the Java projects in your workbench. It provides you with a Java-specific view of the resources shown in the Navigator. The element hierarchy is derived from the project's build class paths.

For each project, its source folders and referenced libraries are shown in the tree. You can open and browse the contents of both internal and external JAR files.

Hierarchy view

The Type Hierarchy view offers three different ways to look at a class hierarchy:

  • Show the Type Hierarchy displays the type hierarchy of a type.
  • Show the Supertype Hierarchy displays the supertype hierarchy of a type.
  • Show the Subtype Hierarchy displays the supertype hierarchy of a type.

Projects view

The Projects view which is shown by default in the Java Browsing perspective shows Java projects and their source folders. The source folders are those defined on the project's Java build path.

For each project, its source folders and referenced libraries are shown in a tree. You can select projects, source folders and both internal and external JAR files but you can not drill into them. The selection is fed into the other views of the Java Browsing perspective.

Packages view

This view which is shown by default in the Java Browsing perspective shows a list of Java packages. The currently selected element is used to feed this view.

Types view

This view which is shown by default in the Java Browsing perspective shows a list of Java types. The currently selected element is used to feed this view.

Members view

This view which is shown by default in the Java Browsing perspective shows the content of a type, compilation unit or class file. The currently selected element is used to feed this view.
