String externalization

For an explanation what string externalization and internationalization are please see:

  • http://www-106.ibm.com/developerworks/opensource/library/os-i18n/?loc=dwmain

  • http://java.sun.com/docs/books/tutorial/i18n/intro/index.html

The JDT uses comments put in code to mark strings as externalized.

These comments are of form //$NON-NLS-n$ where n is the 1-based index of the string in a line of code.

The JDT provides the following support for internationalization and string externalization:

  • Compiler option that lets you mark not-externalized strings as compile-time warnings or errors.

    • See the Window > Preferences > Java > Compiler > Errors and Warnings > Not-externalized strings preference
  • Support to find compilation units that contain not-externalized strings.
  • A wizard that lets you externalize strings.
  • Support for finding unused and incorrectly used keys for strings externalized to property files.
