Java editor

The JDT provides an editor that gives you specific support for editing Java code.

The editor is associated with the Java-specific Outline view, which shows the structure of the active Java compilation unit. It is updated as the user edits the compilation unit.

The Java editor can be opened on binary CLASS files. If a JAR file containing the CLASS files has a source attachment, then the editor shows the corresponding source.

The editor includes the following features:

  • Syntax highlighting
  • Content/code assist
  • Code formatting
  • Import assistance
  • Integrated debugging features
  • Can be configured to either show an entire compilation unit or a single Java element only (to change the setting, use the toolbar button Show Source of Selected Element Only)

By default, this editor is activated by either:

  • selecting Open With > Java Editor from a selection's pop-up menu in the Navigator, Package Explorer or Types view
  • clicking a Java file in the Navigator or Package Explorer view (see the  Window > Preferences > Workbench preference page to choose between single and double click modes)
  • clicking a Java type, method or field in the Package Explorer, Types, Members or Hierarchy view (see the  Window > Preferences > Workbench preference page to choose between single and double click modes)
  • single-clicking a currently-open Java file in the Navigator or Package Explorer view.
  • single-clicking a Java type, method or field (if its compilation unit is currently open) in  the Package Explorer, Types, Members or Hierarchy view
  • selecting Open from the pop-up menu of either a type or a method in the Hierarchy, Members, Types or Package Explorer view.
