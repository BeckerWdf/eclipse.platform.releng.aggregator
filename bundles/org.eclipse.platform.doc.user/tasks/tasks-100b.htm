Merging from a stream

Merging takes changes between the initial state and the end state in a stream, and merges them into your workspace. Typically the initial state is the root of a stream (project version) and the end state is either the stream or another project version.

It is very important to understand that the destination of the merge is always the project in your workbench. After the merge has completed you can test the changes locally and then release them to the new stream.

To merge a project:

  1. In the Repositories view, select the project that you want to merge.

  2. From the project's pop-up menu, select Add To Workspace.

  3. In the Navigator view, right-click the project and select Team > Merge from the pop-up menu. The Merge wizard opens.

  4. In the Merge Setup page, select the project and the repository with which the project is shared, and click Next.

  5. On the Choose Initial State page, choose the initial state of the merge.  This will usually be the root project version on which the originating stream was split.  Select a version in the list and click Next.

  6. On the Choose End State page, choose the end state of the merge.  This can be either a  version (for example if you versioned from your stream before the merge) or it can be a stream.  Click Finish to close the wizard. A merge view opens in the editor area.  This shows the changes between the initial and end state based on the resources in your workspace. 

  7. Catch up or merge changes within this view. At this point, you are working locally. You cannot release changes from within the merge view.

  8. After merging the changes, close the merge view and test and review the result of the merge. If you are happy with it, you can release the changes to the destination stream.

Tip: After merging changes into a stream, it is recommended that you create a project version to save the project's state after the merge. This project version can be used to split again for subsequent merges.
