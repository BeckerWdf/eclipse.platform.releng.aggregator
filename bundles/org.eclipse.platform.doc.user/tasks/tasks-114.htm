Releasing

You can release workbench resources to the stream so that other team members can catch up with your work.  To release changes to a stream:

 

1.

In the Navigator view, select the resource (or resources) that you want to release to the stream.

2.

Bring up the context menu.  Select the "Team > Synchronize with Stream" menu item.

3.

In the Synchronization view, select the Release mode button on the view's toolbar to filter out any modified workbench resources (outgoing changes) you may have.  See Synchronizing.  

4.

If there are no conflicts (red arrows), then you can release all the resources to the stream.  Go to Step 6. 

If there are conflicts, go to step 5.

5.

Resolve conflicts.

This process will likely involve a merge.  Use the text compare area to merge  resources with conflicts.  You can copy changes from the repository version of the resource to the workbench version of the resource and save the merged workbench resource.  Once all the conflicts in the Structure Compare have been resolved, you are ready to release. 

6. Select the top-most resource (or resources) in the hierarchy of the Structure Compare pane, bring up the context menu, and select the "Release" menu item.  

7.

In the Release Comment Needed dialog, provide a comment for this release (e.g., "Fixed the spelling mistakes").
