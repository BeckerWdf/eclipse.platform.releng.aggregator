Releasing resources to the stream

Tip: Before performing your first  synchronize operation, consider which files and directories you wish to ignore at release time, and add those to the appropriate "ignore" facility. Once a resource is under version control, it cannot easily be ignored in subsequent release operations. Resources come under version control in the first place when a synchronize operation is performed on a resource or any of its parent folders. See "Filtering resources from a release."

You can release workbench resources to the stream so that other team members can catch up with your work.  To release changes to a stream:

  1. In the Navigator view, select the resources that you want to release to the stream.

  2. Right-click to open the pop-up menu and select Team > Synchronize with Stream.

  3. Switch to the Team perspective. On the toolbar of the Synchronize view, select the Release mode button to show any modified workbench resources (outgoing changes) that you may have.

  4. If there are conflicts (red arrows), resolve them. Use the text compare area to merge  resources with conflicts. You can copy changes from the repository version of the resource to the workbench version of the resource and save the merged workbench resource.  Once all the conflicts in the Structure Compare area have been resolved, you are ready to release. 

  5. When there are no conflicts, you can release all resources to the stream. In the Structure Compare pane, right-click the top of the hierarchy that you want to release, and select Release from the pop-up menu.

  6. In the Release Comment Needed dialog box, provide a comment for this release (for example, Fixed the spelling mistakes).

Tips: 

  • To configure the workbench to switch automatically to the Team perspective on release and catch-up operations, select Workbench > Preferences > Team from the main menu bar.

  • You can also change the prefix label for outgoing changes that appears in the Navigator view, when the Show Version Info menu item is enabled, on the Team preferences page.
