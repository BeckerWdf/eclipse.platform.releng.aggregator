Stopping resources from releasing at synchronization time

If you place resources under version control by synchronizing them without first defining "ignore" criteria, you may notice that certain files (for example bin/*.obj and *.ba~) appear in the Synchronize view as outgoing additions. You probably do not want these files to be synchronized. You can still ignore these files, despite the fact that the workbench already considers them to be under version control. You first have to delete the files that you want ignored. (This is usually is not a problem since ignored files are either temporary or build results of your important files, the source). Then add the ignore pattern using either the global or CVS ignore mechanism.  At this point you can rebuild your project and synchronize your project with the stream.

Here is how you can ignore files once they already appear as outgoing additions:

  1. From the Navigator view, delete the /bin directory and all the *.ba~ files.

  2. Create a .cvsignore file (File > New > File) in the parent directory of /bin, and add bin to the file.  Save the .cvsignore file.

  3. Select Workbench > Preferences > Team > Ignored Resources and add *.ba~ to the list of patterns to be ignored.

  4. Select the Project > Rebuild All to rebuild the project.

  5. In the pop-up menu of the Navigator view, select Team > Synchronize with Stream to synchronize with the stream. The /bin directory and the *.ba~ files no longer appear as outgoing additions. The .cvsignore file should be released to the stream so that others in your team can share this setting.
