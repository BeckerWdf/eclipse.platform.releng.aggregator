Catching up

While you are working in the workbench, other members of your team may be releasing changes to the stream. To update the resources in your workbench, you may "catch up" with the stream. You control when you choose to catch up with a stream and which incoming changes from a stream you want to accept into the workbench.

When you catch up to a stream, you will see changes that have been released to the stream since you last caught up. If you accept these incoming changes, they are incorporated into your local workbench. If your local workbench contains any outgoing changes that conflict with incoming changes from the stream, then you must merge the differences in your workbench, as described below.

To catch up:

  1. In the Navigator view, select the resources whose stream you want to catch up with.

  2. From the pop-up menu for the selected resources, select Team > Synchronize with Stream.

  3. Switch to the Team perspective.* On the toolbar of the Synchronize view, click the Catch up mode button to filter out any modified workbench resources (outgoing changes) that you may have.

    In the Text Compare area of the Synchronize view, local workbench data is represented on the left, and repository stream data is represented on the right. Click the Show ... Changes buttons in the Structure Compare editor as needed to display all differences.

  4. If there are conflicts (represented by red arrows), resolve them. Use the text compare area to merge  resources with conflicts. You can copy changes from the repository version of the resource to the workbench version of the resource and save the merged workbench resource. 

  5. Once there are no conflicts, you can safely catch up all the  resources in the workbench without having to worry that you will lose changes you made in the workbench. In the Structure Compare pane, right-click the top-most resources that you want to catch up with, select Catch Up from the pop-up menu.

The stream itself is not changed when you catch up. When you accept incoming changes, these changes are applied to your workbench. The stream is only changed when you release your outgoing changes.

*Tip: To configure the workbench to switch automatically to the Team perspective on release and catch-up operations, select Workbench > Preferences > Team from the main menu bar.
