Associating a project with a stream

When you change the sharing of a project, you change the stream with which it is associated.

Important: If you change the sharing to a stream in another repository, your project's existing synchronization information is deleted. (This information is used to track the version state of local resources and helps determine outgoing, incoming and conflicting changes.) This is also true if  you add the project from the stream via the Repositories view. 

By contrast, changing the sharing to another stream in the same repository does not change the synchronization information. You would normally do this task in order to split from a stream and release the changes into another stream. 

To change the stream that a project is associated with:

  1. In the Navigator view, select the project whose stream association you want to change.

  2. Select Properties from the project's pop-up menu. The Properties page opens.

  3. In the left pane, select Team. The Team Properties page opens.

  4. Click the Change button to open the Set Project Sharing window.

  5. Select the repository and stream with which you want to associate the project, then click OK.

  6. Click OK to close the Properties page.
