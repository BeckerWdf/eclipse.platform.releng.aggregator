Splitting from a stream

Typically, you will want to split a stream under the following conditions:

  • To fix a bug in an old version of a project

  • To store project variants or private work in a separate stream

When splitting a stream you always split at a predefined point in time. This is called the initial state of the new stream. When you split a stream, the initial state is always a version. 

Generally, you will split a stream with the intention of eventually merging the changes back into the originating stream.

There are two options for splitting a stream.

Option 1: Splitting based on a project version

Choose this option if you are splitting based on an existing project version.  This is often done to fix a bug in a previous version of your project.

  1. Switch to the Team perspective or add the Repositories view to the current perspective.
  2. From the pop-up menu for the Repositories view, select New > Stream.. The New Stream wizard opens.
  3. Select the repository that contains the stream you want to split.
  4. Enter a name for the new stream and click Finish to close the wizard.
  5. Add a project version to the new stream to set the initial state of the stream. To do this, select the new stream in the Repositories view and select Copy Version to Stream from its pop-up menu. The Choose Versions window opens.
  6. Select the projects and their versions that you wish to copy to the stream. 
  7. Expand the new stream to see the projects. You will see the resources that are in the project version that you added to the stream.
  8. Select the project that you want to add to the workbench.  From its pop-up menu, select Add to Workspace.  Changes released from the project will now go to the new stream.

If you enable Show Version Info via the drop-down menu in the Navigator view's toolbar, you will see the new stream name shown to the right of the project.

Option 2: Splitting with changes in your workspace

In some cases you may not want to release your changes to everyone in the team, but would still like to version them. You may want to do this if your changes are not stable yet or if it will take a long time before you are ready to release them to the team. In this case you will want to create a new stream for releasing your changes.

  1. Create a version of your project from the workspace to capture the state before you made any changes.  This version will contain the base versions of the project's resources.  The changes you have made from the base versions will later be released to the new stream.

    Right-click your project in the Navigator view and select Version from Workspace from its pop-up menu.

  2. Complete the Version Selected Resources page. (Versioning is covered in a different topic in this documentation.) 
  3. When prompted, review the outgoing changes listed in the details window. 
  4. Switch to the Team perspective or add the Repositories view to the current perspective. Right-click in the Repositories view and select New > Stream from the pop-up menu to create a new stream. The New Stream wizard opens.
  5. Select the repository that contains the stream you want to split.
  6. Enter a name ofr the new stream and click Finish to close the wizard.
  7. Add the project version to the new stream to set the initial state of the stream. To do this, select the new stream in the Repositories view, select Copy Version to Stream from its pop-up menu, and in the Choose Versions window select the project and version that you created earlier.
  8. In the Navigator view, select the project and select Properties from its pop-up menu to change sharing of the project to the new stream.
  9. Synchronize the project with the new stream. Your changes will appear as outgoing to the new stream. Release them into the new stream.
