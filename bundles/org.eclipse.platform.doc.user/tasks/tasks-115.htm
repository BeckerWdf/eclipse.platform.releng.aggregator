Synchronizing with a stream

Tip: Before performing your first  synchronize operation, consider which files and directories you wish to ignore at release time, and add those to the appropriate "ignore" facility. Once a resource is under version control, it cannot easily be ignored in subsequent release operations. Resources come under version control in the first place when a synchronize operation is performed on a resource or any of its parent folders. See "Filtering resources from a release."

To synchronize resources in the workbench with resources in a stream:

  1. In the Navigator view, select the resources that you want to synchronize with a stream.

  2. Right-click and select Team > Synchronize with Stream. The Synchronize view opens.

  3. Use the toolbar buttons to switch modes for this view. There are three:

    • Catch up mode - shows incoming changes only (resources in the stream that differ from what is in the workbench). Refer to the "Catching Up" topic.

    • Release mode - shows outgoing changes only (resources modified in the workbench). Refer to the "Releasing" topic.

    • Catch Up/Release mode - shows both incoming and outgoing changes.

    Important: It is preferable to catch up resources in the workbench first, resolve any conflicts that exist by merging, then release workbench resources to the stream.

  4. To show only conflicts (resources edited in both the workbench and in the stream), click the Show Only Conflicts button on the view's toolbar.

The resources displayed in the Structure Compare area will depend on which filters are used in the Synchronize view. In the Text Compare area, the workbench resource is on the left side, and the stream resource is on the right side.
