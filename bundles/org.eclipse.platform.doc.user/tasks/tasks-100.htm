Creating a version of a project

Prerequisite: The project that you want to version must first be associated with a stream.

Tip: It is often preferable to synchronize resources with the stream before versioning.

To make a version of a project:

  1. In the Navigator view, select the project that you want to version to the repository.

  2. From the project's pop-up menu, select Team > Version from Workspace. The Version Selected Resources page opens.

  3. Select the version name option that you want to enable for the selected project. (You can only choose the version label at the project level. Other types of resources are versioned when released.)
  4. Click OK to close the Version Selected Resources page and create the version of your project.

Tips: 

  • You can change the initial version label that appears in the Version Selected Resources page by selecting Workbench > Preferences > Team > Initial version name. Type the alphanumeric label that you want to use as the initial automatically generated version label for projects that are versioned for the first time.
