Picking up pending changes

Whenever the Eclipse platform is started it detects and notes any changes that have been made to the installed base of files such as the appearance (or disappearance) of feature and plug-in versions. New or updated features are not automatically configured for an existing workspace; rather, the user is told on start up that there are updates and given the opportunity to configure them. If accepted, the new and updated feature versions become a part of the feature and plug-in configuration used for that workspace. If not accepted, the new and updated feature versions remain dormant (disabled) for that workspace, but available to be picked up at a later time.

  1. Click Yes when informed of updates and prompted to enter the update manager. Or click Help > Software Updates > Pending Changes.
    The Configuration Changes page opens showing a list of detected changes.
  2. Select and expand detected change items to see the associated feature versions.
  3. Check items that are to be included in the configuration. Unchecked items are ignored (but remain pending).
  4. Click Finish to update the configuration, or Cancel to retain the current configuration (pending changes remain pending).
  5. Click Yes when asked to exit and restart the workbench for the changes to take effect.
