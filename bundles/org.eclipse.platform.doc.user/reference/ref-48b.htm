Automatic version labeling schema

If you select the Automatic version labeling format in the Version Selected Resources page, the following "guessing" schema is used to assign version labels.

  • If a project has no previous version label, then the default version name is used.  You can indicate the desired default version name.
  • If a project already has a previous version label:
    • The previous version label is separated into a text prefix and a numerical suffix, where the suffix is any trailing digits on the previous version name. If the previous version has no trailing digits, then the suffix is 0.
    • The first guess at a version label is the text prefix combined with the incremented suffix (that is, suffix + 1).
    • If the guess matches an existing version label for that project, the suffix is incremented again to produce another guess.  This continues until a guess is found that does not match any previous version.

Tip: You can change the initial version label that appears in the Version Selected Resources page by selecting Workbench > Preferences > Team > Initial version name. Type the alphanumeric label that you want to use as the initial automatically generated version label for projects that are versioned for the first time.
