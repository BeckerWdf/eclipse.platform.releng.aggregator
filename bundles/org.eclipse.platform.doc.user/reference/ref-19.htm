Team

The Team preferences page contains options which affect the version management Team support:

Option

Description

Default

Initial Version Name

This is the version name that will be suggested as the first (initial) version of a project.

v1

Label prefix for outgoing changes

This label is used in the Navigator view to indicate which resources have changed since the last time synchronization occurred.  To show this label, enable  Show Version Info in the Navigator or Package view's tool bar drop down view menu.

>

Perform refresh automatically before synchronize view opens

Before you synchronize (e.g., catch up with or release to a stream), the workbench will do a refresh operation to ensure that resource information is current.

On

Switch to team perspective on catch up/release

When synchronizing resources with a stream, the workbench will automatically change to the Team perspective.

On

Prompt for a location when adding projects from the repository

When you add projects to the workbench from a repository, you will be prompted for a location for those projects.

Off

Use Catch Up/Release mode when synchronizing

The Synchronize view can open in catch up mode, release mode, or catch up/release mode.  Turning this option on will open the Synchronize view in catch up/release mode, regardless of what you are doing (e.g., catching up, releasing, or both).

Off
