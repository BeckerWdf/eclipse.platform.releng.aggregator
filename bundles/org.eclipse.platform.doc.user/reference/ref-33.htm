Synchronization View

This view allows you to catch up resources in the workbench and release resources from the workbench to a stream.  

Here is what the Synchronization view looks like:

There are 3 different viewing modes (filters) you can use in this view to make the synchronization process easier:

Catch up:  In this mode, only resources which have versions more recent than the base version of resources in the workbench (incoming changes) will be visible.

Release:  In this mode, only resources which have been modified in the workbench (outgoing changes) will be shown.  

Catch up/Release:  In this mode both incoming and outgoing changes will be shown.  You can also choose to filter out incoming and/or outgoing changes from the drop down menu in this mode.  

In any of these viewing modes you can choose to show only resource conflicts.  You can also choose to Ignore Whitespace differences or Refresh From Repository from the dropdown menu of this view.

It is preferable to catch up resources in the workbench first, resolve any conflicts that exist by merging, then release workbench resources to the stream.

Structure Compare Pane

This pane allows you to view a hierarchy tree of all the resource differences between the stream and the workbench.

Text Compare Pane

This pane allows you to view the content differences of the resource selected in the structure compare pane .
