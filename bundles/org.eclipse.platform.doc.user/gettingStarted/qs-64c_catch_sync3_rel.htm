Releasing

We will now use the Release Mode of the Synchronize view to release the one remaining outgoing change. 

1.

Place the Synchronize view into release mode by clicking on the Release Mode button (#1) and confirming that the title of the view now indicates we are in Release Mode (#2).

 

2. Select file3.txt (the outgoing change). 

3.

The comparison viewer shows that our copy of file3.txt has a change to the second line.  The repository copy is shown in the right pane.

4.

From the context menu of file3.txt choose Release. Be sure to enter a meaningful description when prompted for one. 

Your workbench copy is now released into the stream.

5.

Select the Resource perspective from the shortcut bar at the far left of the window.
6. At this point Fred may want to Synchronize with the stream to catch up to the changes you just released.

We have successfully collaborated with Fred by making simultaneous changes to the same project. The workbench's Synchronization view  and Repository History view allowed us to determine the specific changes each user was making and to quickly stay on top of the ongoing changes being released into the stream.
