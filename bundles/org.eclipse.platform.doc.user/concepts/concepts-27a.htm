Ignoring Resources from Version Control

Often a user may not want to commit all resources in their project to the repository.  For example, external editors may create temporary files in your project, compilation of .java files creates .class files, and likewise many build operations result in binary files. These files, when taken together, may be quite large.  In addition, they may be re-generated whenever a build is performed, resulting in many  outgoing changes. Typically these are not files that one wants to share with other members of a team or persist in the repository.  

Eclipse tools will mark there output as being "derived" resources and repository providers that are integrated with the workbench, such as CVS, will ignoe these resources. Eclipse also provides a Global Ignore mechanism for specifying a set of file name patterns which identify resoures which are to be ignored. Individual repository providers may also have their own mechanism that work in conjunction with the global ignore list. For instance, CVS has a facility which allows the user to specify file name patterns for resources to be ignored in a special file named .cvsignore.
