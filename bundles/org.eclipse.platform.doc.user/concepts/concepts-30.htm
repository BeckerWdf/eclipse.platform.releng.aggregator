Team synchronization

In the team programming environment, there are two distinct processes involved in synchronizing resources: catching up with the stream and releasing to the stream. 

When you make changes in the workbench, the resources are saved locally.  Eventually you will want to release your changes to the stream so others can have access to them.  Meanwhile, others may have released changes to the steam.  You will want to catch up by updating your workbench resources with their changes.

Important: It is preferable to catch up before releasing, in case there are conflicts  with the resources in your workbench and the resources currently in the stream.  

When you select Team > Synchronize with Stream for a resource, the Synchronize view opens. You can set filters to control whether you want to view only incoming changes or outgoing changes. Incoming changes come from the stream; if accepted, they will update the workbench resource to the latest version currently released into the stream. Outgoing changes come from the workbench; if released, they will update the stream resources to the version currently present in the workbench.

Regardless of which mode (filter) you select, the Synchronize view always shows you conflicts that arise when you have locally modified a resource for which a more recent version is available in the stream.  In this situation you can choose to do one of three things: catch up the resource from the stream, release your version of the resource to the stream , or merge your work and the stream resource.  Typically you will want to merge, as the other two options will result in loss of work.
