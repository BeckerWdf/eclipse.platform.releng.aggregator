Creating component JARs

Unless you take responsibility for maintaining the build script yourself, it is best to regenerate new scripts each time before executing them.  This way, changes in folder structure, new files, deleted files, and new plug-ins will be accounted for.

When Ant scripts are executed from within the wizard, a wizard console will appear below the checkboxes so that you can see build messages. 

When a build is complete, you should see individual JARs in the plug-in and fragment projects, as well as the component JAR that has the following name format: <componentId>_<component_version>.jar

You can also run the build by selecting the component build.xml file and selecting the "Run Ant" action from the popup menu. In this case, build messages will be shown in the progress indicator dialog. The resulting JARs will be identical in both cases.
