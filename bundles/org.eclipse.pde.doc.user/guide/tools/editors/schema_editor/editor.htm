About Extension Point Schemas

An extension point schema is a valid W3C XML schema file that defines a contract for extension points that extensions must conform to in order to plug-in and extend the platform. More specifically, an extension point schema specifies a formal grammar for an extension point's elements and attributes that govern the structure, content and semantics of an extension's declarative XML mark-up.

Extension point schemas are used during plug-in development to do the following:

  • Validate all extensions defined in workspace plug-ins.
  • Generate reference extension point documentation.
  • Provide user assistance in creating extensions.

About This Editor

The Extension Point Schema Editor allows you to do the following:

  • Create and edit an extension point schema's grammar.
  • Document an extension point schema's grammar.
  • Specify useful metadata used by PDE tooling.
  • Preview the extension point schema's reference document.

This editor abstracts the extension point schema file mark-up as an implementation detail you do not need to learn.
