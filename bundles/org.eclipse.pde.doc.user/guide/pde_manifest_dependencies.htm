Dependencies page

The Dependencies page shows the dependencies that your plug-in has on other plug-ins.  All the plug-ins on which your plug-in can depend are presented in a checkbox list.  Checking any of the plug-ins in this list will update the plug-in manifest and configure the build path of the project so that the classes of the newly added plug-in are visible to the plug-in under development.

Since the PDE distinguishes between workspace and external plug-ins, the eligible plug-ins will be shown in two separate lists of checkboxes. 

As you establish dependencies on other plug-ins, a dependency graph will illustrate the dependencies. Since plug-ins you depend on might depend on other plug-ins, you can traverse the dependency tree by clicking on the nodes and also using the buttons above the graph in the same way you would use Web browser buttons. 

You can also open the manifests for other plug-ins by selecting Open from the pop-up menu.

In addition to showing plug-ins that your plug-in depends on, the graph shows plug-ins that depend on your plug-in. To see this information, select the radio button "Show plug-ins that require this plug-in". 

Since our plug-in has just been created in this example, and there are no other plug-ins in the workspace, this graph will be empty. To see something interesting, select an external plug-in like "Eclipse Workbench" and then select the radio button. You should see all the plug-ins that depend on the workbench.
