Running a plug-in

As you develop your plug-in in the workspace, the incremental Java compiler will compile your Java source code and place the *.class files into the bin directory of your PDE project. When you are ready to test your plug-in, you can launch a run-time platform instance to test your new plug-in.

The JDT tools allow you to use the external JDK of your choice. PDE uses the same information that the JDT tools use, so you need to ensure that your default VM and other runtime information is defined.  See the JDT User Guide for more information on this process.

The JDT launching tools require that a runnable class (a class that has a main method) is selected in order to run a program.  PDE supplies its own launcher that works differently.  You only need to select a PDE project in order to run your plug-in.  The PDE launcher (also called the "Runtime Workbench") will use the runtime platform path setting in the PDE Preferences to start another instance of the platform and include the plug-ins in your workspace. This way, you can test your work without exporting or packaging the plug-ins.

You can run your plug-in by pressing the "Run" icon on the toolbar.  (The PDE launcher is set up as the default launcher for projects created by PDE.)  The Run/debug perspective will open automatically and a new platform instance will soon follow.

Example:  Running the Action Set

Before running, make sure that your environment is set up as described in Configuring PDE. You will want to have different workspaces for design and run-time.

Select the com.example.xyz project we created earlier and and press the Run button. Another platform instance should open. Our new action set is not immediately visible, so we need to activate it by selecting Perspective->Customize->Other... and selecting the checkbox for "Sample Action Set" (the action set that we added). "Sample Menu" should now show up in the menu bar with the "Sample Action" item. Selecting it should output "Hello, PDE world" to the Console view of the Debug perspective (in the host platform instance).
