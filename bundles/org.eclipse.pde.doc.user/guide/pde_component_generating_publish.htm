Publishing a JAR

PDE does not provide automated publishing support for components.

To publish a component, you need to place the install.xml and the component JAR on a platform update server.

See Platform Installation and Update for a description of the update mechanisms and directory structures expected.

Let us assume that there is a server that your team uses for serving platform components. The file system structure would look similar to this example:

   <server_root>/
      install/
         components/
            install.index
            org.eclipse.platform_1.0.1/
            org.eclipse.jdt_2.9.5/
            org.eclipse.pde_1.0.4/

The file install.index in this example would contain:

   org.eclipse.platform_1.0.1
   org.eclipse.jdt_2.9.5
   org.eclipse.pde_1.0.4

You simply take your component manifest and the JAR and place them in a directory that has the same name format as the JAR itself minus the .jar extension. You need to add an entry in the install.index file because Update Manager cannot simply iterate through the directories when accessing the server using HTTP protocol.

If everything is done correctly, products built on the platform can use Update Manager to point to your site, install your component, and restart.  When they restart the platform, your plug-in written in PDE should be available.

As you continue work on your component, you can create new versions and periodically publish them on the update server. Just don't forget to increment version numbers.
