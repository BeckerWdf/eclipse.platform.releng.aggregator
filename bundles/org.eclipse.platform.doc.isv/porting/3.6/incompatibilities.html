Incompatibilities between Eclipse 3.5 and 3.6

Eclipse changed in incompatible ways between 3.5 and 3.6 in ways that affect plug-ins. The following entries describe the areas that changed and provide instructions for migrating 3.5 plug-ins to 3.6. Note that you only need to look here if you are experiencing problems running your 3.5 plug-in on 3.6.

  1. SWT's Event#stateMask includes location

1. SWT's Event#stateMask includes location

What is affected: Clients that use Event#stateMask or KeyEvent#stateMask

Description: The stateMask for key events has been extended to include a location mask for keys that can appear multiple times on the keyboard (e.g. left and right Ctrl, or numbers on the keypad). This change affects:

  • clients that compare stateMask to a modifier constant without using SWT.MODIFIER_MASK. The recommended pattern for that case is: 
    boolean isMod1 = (event.stateMask & SWT.MODIFIER_MASK) == SWT.MOD1;
  • clients that compare stateMask to 0 to determine if no modifiers and no buttons are down, or any modifier or any button is down. The recommended pattern for that case is: 
    int mask = SWT.MODIFIER_MASK | SWT.BUTTON_MASK;
boolean noModifierAndNoButton = (event.stateMask & mask) == 0;
boolean anyModifierOrAnyButton = (event.stateMask & mask) != 0;

Action required: Clients that read Event#stateMask should make sure they use appropriate masks when comparing the stateMask.
