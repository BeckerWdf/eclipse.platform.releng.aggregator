Targets

Identifier: org.eclipse.core.target.targets

Description: The workspace supports the notion of incrementally synchronizing with resources located outside the workspace (i.e., in targets).  A target might be a web server, embedded device, remote filesystem, ... depending on the application . By default the platform supports WebDAV and filesystem based uni- and bi-directional targets.  The set of targets available to users can be extended using the targets extension point.

The targets extension-point allows target writers to register their target implementation under a symbolic name that is then used from within the workspace to find and run targets. The symbolic name is the id of the target extension.

Configuration Markup: 

   <!ELEMENT target run?>
   <!ATTLIST target
      id             CDATA #REQUIRED
      name           CDATA #REQUIRED
   >

  • id - a unique name that can be used to identify this target
  • name - a translatable name that may be presented to the user to identify this target
   <!ELEMENT run parameter*>
   <!ATTLIST run
      class          CDATA #REQUIRED
   >
  • class - the fully-qualified name of an implementation of org.eclipse.core.target.ITarget.
   <!ELEMENT parameter EMPTY>
   <!ATTLIST parameter
      name         CDATA #REQUIRED
      value        CDATA #REQUIRED
   >
  • name - the name of this parameter made available to instances of the specified target class
  • value - an arbitrary value associated with the given name and made available to instances of the specified target class

Examples: 

Following is an example of a target configuration:

   <extension point="org.eclipse.core.target.targets">
      <target id="com.example.coolTarget" name="Cool Target">
         <run class="com.example.targets.Cool"/>
      </target>
   </extension>

API Information: The value of the class attribute must represent an implementation of org.eclipse.core.target.ITarget.

Supplied Implementation: The platform supplies several targets including uni- and bi-directional WebDAV and filesystem based targets.
