Boolean expressions and action filters

When a plug-in contributes an action to the workbench UI using one of the menu extension points, it can specify conditions under which the menu item is enabled (or visible) in the menu.  In addition to supplying simple enabling conditions such as selection counts and selection classes, plug-ins can use boolean expressions that give more refined control over the enablement of an action.

Boolean expressions

The boolean expressions can contain simple boolean operators (NOT, AND, OR) and predefined expressions that evaluate the following conditions:

  • objectClass - true if each object in the selection subclasses or implements the class.

  • objectState - true if the named attribute meets the specified value.  IActionFilter assists in evaluating the expression.  An action filter dynamically computes the enablement criteria for an action based on the target selection and the value of named attributes.

  • systemProperty - true if the named system property meets the specified value.

  • pluginState - specifies whether the specified plugin (by id) should be installed or activated

See action expressions for samples of these expressions and a complete description of the XML.
