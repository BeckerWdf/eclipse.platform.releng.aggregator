Creating the plug-in project

You can use any Java IDE you wish to build Eclipse plug-ins, but we'll walk through the steps for building our plug-in with the Eclipse Java IDE, since this is the typical case.  If you are not already familiar with the Eclipse workbench and the Java IDE, consult the Java Development User Guide for further explanations of the steps we are taking.  For now we are focusing on the code, not the tool.  However, there are some IDE logistics for getting started.

Setting up your environment

If you are starting with a clean workspace, you will need to ensure that your code has access to the plug-in org.eclipse.ui.ide. This is the plug-in that supplies the default platform workbench. Import this plug-in and any plug-ins it requires into your run-time workbench instance by performing the steps below. If you already have these plug-ins loaded in your workspace, proceed to the section "Creating your plug-in project".

  1. From the resource perspective, use File > Import... > External Plug-ins and Fragments.
  2. Continue clicking on the Next > buttons until you get to the screen called Selection. This screen will contain a list of all of the plug-ins in your host workbench instance.
  3. Select org.eclipse.ui.ide and then click the button Add.
  4. Click on the button Required Plug-ins. The list on the right should fill up with all the plug-ins that org.eclipse.ide requires.
  5. Click on the Finish button. Your Navigator view should contain org.eclipse.ui.ide and all of the plug-ins it requires (such as org.eclipse.ui, org.eclipse.swt, etc.).

For a complete discussion of setting up host and run-time workbench instances consult the PDE Guide.

Creating your plug-in project

You will need to create a project that contains your work. We'll take advantage of some of the code-generation facilities of the PDE to give us a template. This will set up the project for writing Java code and generate a default plug-in manifest file (explained in a moment) and class to hold our view.

  1. Open the New Project... wizard (File > New > Project...) and choose Plug-in Project from the Plug-in Development category.  Use org.eclipse.examples.helloworld as the name for your project and click Next.
  2. On the Plug-in Project Structure page, check the box for Create a Java project (this should be the default) and click Next to accept the default Java project structure.
  3. On the Plug-in Content page, look at the default settings. The wizard sets org.eclipse.examples.helloworld as the id of the plug-in.  The wizard will also generate a plug-in class for your plug-in and allow you supply additional information about contributing to the UI. These defaults are acceptable, so click Next.
  4. On the Templates page, check the box for Create a plug-in using one of the templates. Then select the Plug-in with a view template. Click Next.
  5. We want to create a minimal plug-in, so at this point we need to change the default settings to keep things as simple as possible. On the Main View Settings page, change the suggested defaults as follows:
    • Change the Java Package Name from org.eclipse.examples.helloworld.views to org.eclipse.examples.helloworld (we don't need a separate package for our view).
    • Change the View Class Name to HelloWorldView.
    • Change the View Name to Hello View.
    • Leave the default View Category Id as org.eclipse.examples.helloworld.
    • Change the View Category Name to Hello Category.
    • Leave the default viewer type as Table viewer (we will change this in the code to make it even simpler).
    • Uncheck the box for Add the view to the resource perspective.
    • Click Next to proceed to the next page.
  6. On the View Features page, uncheck all of the boxes so that no extra features are generated for the plug-in. Click Finish to create the project and the plug-in skeleton.
  7. When asked if you would like to switch to the Plug-in Development perspective, answer Yes.
  8. Navigate to your new project and examine its contents.

The skeleton project structure includes several files and a Java package. The important files at this stage are the plugin.xml (manifest) file and the Java source code for your plug-in. We'll start by looking at the implementation for a view and then examine the manifest file.
