Building a team UI

The team UI plug-in is based on the philosophy that the team subsystem should provide simple hooks and slots for providers to add their UI implementation.  Rather than define standard team operations, the team plug-in leaves the workflow up to the provider implementation.  All it does in the way of defining actions and preferences is to define a "Team" slot where you can add your specific UI elements.

We've seen that some of the core team extensions, such as ignored files, file types, and project sets, influence the UI via the preferences dialog.  Otherwise, you must build your own UI by designing the actions that your repository supports and in what views and editors these actions are accessible.  The implementation for your UI is done with the standard workbench extension points.  In addition, you can define your own preferences which plug in to the standard framework.  You may even need to build your own views to present repository specific information and operations. (The CVS client implements its own views for synchronizing with the repository and navigating the repository.)
