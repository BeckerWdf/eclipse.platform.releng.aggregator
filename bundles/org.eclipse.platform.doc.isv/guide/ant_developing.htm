Developing Ant tasks and types within Eclipse

To successfully develop Ant tasks and types within Eclipse some guidelines must be followed:

  • Plug-ins that plan to contribute Ant tasks or types need to have separate source folders for the Ant contribution source if the developer wishes to test/debug using the Run-time workbench support.
  • These separate source folders must have separate output locations.
  • When testing/debugging the new Ant tasks or types, the the Run-time workspace launch configuration must be configured to not include these output locations in the classpath. This is achieved by not including the Ant output directories in the listing of the Classpath Entries (-dev) on the Arguments tab of the Run-time Workbench launch configuration.
