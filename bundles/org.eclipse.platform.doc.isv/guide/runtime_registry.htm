Extension points and the registry

We've seen that bundle API can be used to find out what bundles are installed on the system, what bundles they require, etc. This information may be important for tools that manipulate the runtime, but a more common case is to find out what extension points have been made available by plug-ins and what extensions are contributed by plug-ins. For example, when the workbench shows the list of available views, it needs to know about all views that might have been registered by plug-ins, so that it can show the views in the list without having to start up each plug-in.
