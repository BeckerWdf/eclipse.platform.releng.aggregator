Tooling platform

The Eclipse platform has been designed from the ground up for building integrated web and application development tooling. While the most basic plug-ins of the platform have been designed so that they can be used independently of any specific tooling paradigm, (Rich Client Platform), the power of Eclipse comes from the variety of plug-ins that build on top of this basic structure.

An important feature for the tooling (IDE) platform is the ability to manage files located in the local file system. Many of the plug-ins supplied in the Eclipse platform are concerned with the creation and manipulation of data stored in files in a workspace managed by the platform. These resources can be viewed, edited, searched, compared, and combined together using a build infrastructure. Plug-ins that supply resource-oriented function are considered part of the IDE tooling platform. This includes the concrete implementation of the IDE workbench, including the resource navigator and the ability to edit files.

Additional plug-ins, such as the help facility, supply function that is not specifically related to resource management. These plug-ins can be added to any configuration of plug-ins, whether the application is designed for IDE tooling or some other purpose.

The following subsystems can be used to add additional functionality to an Eclipse application.

Resource management

The resource management plug-in defines a common resource model for managing the artifacts of tool plug-ins. Plug-ins can create and modify projects, folders, and files, as well as define specialized types of resources.

Resources overview provides an overview of the resource management system.

Additional plug-ins enhance the functionality of the basic resource model. Search support provides facilities for searching the contents of resources and supplying customized UI for displaying search results. Compare support provides facilities for showing two-way and three-way comparisons of resources.

IDE UI

The IDE UI plug-in implements the platform workbench. It defines the standard layout and configuration of the workbench and adds specialized views, editors, and additional extension points for viewing and manipulating resources.

Plugging into the workbench introduces the workbench UI extension points and API.

Team support

The Team plug-ins allow other plug-ins to define and register implementations for team programming, repository access, and versioning.  The Eclipse SDK includes a CVS plug-in that uses the team support to provide CVS client support in the SDK.  

Team support is described in Team support.

Debug support

The Debug plug-ins allow other plug-ins to implement language specific program launchers and debuggers. 

Debug support is described in Program debug and launching support.

Help System

The Help plug-in implements a platform optimized help web server and document integration facility.  It defines extension points that plug-ins can use to contribute help or other plug-in documentation as browsable books.  The documentation web server includes special facilities to allow plug-ins to reference files by using logical, plug-in based URLs instead of file system URLs.

Additional features are provided for integrating help topics in product level documentation configurations.

The help facility is described in Plugging in help.

Product model

The Eclipse platform also provides a structured framework for defining a branded product that is developed using the platform. Plug-in developers can customize many aspects of the platform, including splash screens, window images, and other graphics. Products can be updated dynamically using a remote installation server.

The product model is described in What is a product.
