FormLayout

FormLayout lays out widgets using an attachment model.  A FormAttachment can be defined for each side of a widget and stored in its layout data (FormData).  The FormData can be used to designate a requested width and height for the widget, provided these values don't conflict with the attachment constraints.

A FormAttachment is used to attach a designated side of the widget to the parent Composite or another widget in the layout.  Typically, you do not set attachments on all sides of a widget.  It is very common to specify only one horizontal (left or right) attachment and one vertical (top or bottom) attachment and allow the widgets to take the size specified in their FormData, or their preferred size, if no size is specified in the FormData.

 Attachments can be configured in a variety of ways:

Attachment Type

Usage

position, offset

Attach the specified side of a widget to a position (percentage value 0-100) in the parent widget.  An optional pixel offset can be added.

adjacent widget, offset

Attach the specified side of a widget to the adjacent side of a target widget inside the composite.  An optional pixel offset can be added.  The side where the target widget is attached is inferred based on the side specified in the source widget.  For example, attaching the top edge to a widget implies that it is being attached to the bottom edge of the target widget.

reference widget, offset

Attach the specified side of a widget to a particular side of a target widget inside the composite.  An optional pixel offset can be added.  This can be used, for example, to align the top of a widget with the top of another widget so that they will always line up vertically.
reference widget, centered, offset Center the widget relative to a target widget.  The center need only be specified for one side (left or right, top or bottom) in the direction being centered.
