JDT user interface

The org.eclipse.jdt.ui plug-in implements the Java specific user interface classes that manipulate Java elements. The following roadmap outlines the major functions in the org.eclipse.jdt.ui package and its sub packages.

  • org.eclipse.jdt.ui the package provides support classes for presenting Java elements in the user interface. The class JavaUI is the main access point to the Java user interface components. It allows to programmatically open editors on Java elements, open a Java or Java Browsing perspective and open package and type prompter dialogs. The package exposes constants for retrieving Java user interface parts from the workbench registry and to retrieve preference settings from the Java preferences. JavaUI gives access to the IWorkingCopyManager . It manages the working copies that are in use for Java compilation units.
     
    The interfaces ITypeHierarchyViewPart and IPackagesViewPart define the programming interface to interact with the type hierarchy and packages view parts.

    The classes StandardJavaElementContentProvider and JavaElementLabelProvider support presenting Java elements in JFace viewers. JavaElementSorter can be plugged into a JFace viewer to sort Java elements according to the Java UI sorting style.

    The class JavaElementImageDescriptor can be used to create an image based on an arbitrary base image descriptor and a set of flags specifying which Java specific adornments (e.g. static, final, synchronized, ....) are to be superimposed on the image.

  • org.eclipse.jdt.ui.actions the package provides actions and action groups to populate tool bars, global menu bars and context menus with JDT specific functionality. It is recommended that clients use action groups to populate menus and tool bars instead of adding actions directly. This shields clients from missing newly added actions or from using outdated menu structures.

  • org.eclipse.jdt.ui.jarpackager the package provides classes and interfaces to generate a JAR file. It offers hooks to extend JarPackageData to load and store the JarPackageData and to write the JAR itself. 

  • org.eclipse.jdt.ui.text the package provides support classes for presenting Java text. The class JavaTextTools provides access to the different scanners to partition and color Java source code. The class JavaSourceViewerConfiguration defines a standard configuration for a Java source viewer. 

  • org.eclipse.jdt.ui.text.java.hover the package provides means for presenting text hovers in Java editors. The interface IJavaEditorTextHover defines the programming interface for a text hover. See extension point org.eclipse.jdt.ui.javaEditorTextHovers for how to define a new Java text hover.

  • org.eclipse.jdt.ui.wizards the package provides wizard pages for creating Java elements. Supported elements are projects, packages, classes and interfaces. The class  JavaCapabilityConfigurationPage supports editing the Java build settings (source folder setup, referenced projects, and used and exported libraries).
